## 4. Phase 4 关键技术决策与凭据（接手 AI 必看）

下列三条是本会话用了大量 owner-back-and-forth 才落定的，接手 AI 不要轻易回退。

### 4.1 `obwActual` 改 peak-relative thresholding（默认 `-3 dBc`）

**问题缘起**：Phase 4 引入 C8 metric `ExecutionVsMeasuredBwAbsRelDiffP95 = abs(Execution - SourcePlane) / Execution`，比较的是：
- `Truth.Execution.ModulatedBandwidthHz` —— 在 **clean modulator 输出端** 用 `obwActual` 算的
- `Truth.Measured.SourcePlane.OccupiedBandwidthHz` —— 在 **信道后、合路前、receiver RF 链前**（`comp.Signal`）用 `obwActual` 算的

target < 0.03，但初版用 noise-floor 去噪法（`NoiseFloorPercentile=10` / `NoiseFloorMargin=5.0`）跑出 0.1524。

**根因**：clean 信号噪声底极低（`-100 dB` 量级），noisy 信号噪声底为可测（`-30 dB` 量级）→ noise-floor + margin 给两端不同阈值，对 RRC QPSK 这类滚降信号尾部能量被 clean / noisy 不一致地纳入 OBW 计算，结果 12 % 偏差。

**诊断脚本**：`tools/phase4/diag_phase4_rrc_obw.m`（接手 AI 想验证可重跑）

**最终方案**：peak-relative 阈值（默认 `-3 dBc`）—— 不管 clean / noisy，都把"峰值往下 3 dB"作为统一阈值。在 SNR ≥ 6 dB 时 RRC QPSK / OFDM 双场景下 clean / noisy OBW 偏差 < 5 %。

**接手 AI 警告**：
- 不要回到 noise-floor 去噪法。
- 不要把 `-3` 改大（如 `-10`）—— 会把信号外的噪声纳入，OFDM 场景偏差 > 100 %（实测 8.42×）。
- 不要把 `-3` 改小（如 `-1`）—— 主瓣边沿衰减还没到 1 dB，会切掉真正的占用带宽。
- C8 SNR floor `SnrFloorDb=6.0` 是配套的（在 `test_baseline_sweep_200.m::localScoreSource` 内）—— 排除 SNR < 6 dB 样本。两个共同保证 baseline-v0 上 0.0212 < 0.03。

**官方文档参考**：[obw（Signal Processing Toolbox）](https://www.mathworks.com/help/signal/ref/obw.html)

### 4.2 C9 P95 wallclock budget 由 45.0 s 上修到 47.0 s

**问题缘起**：Phase 3 baseline `WallclockSecPerScenarioP95=41.53s`，初版 +8 % budget = 45.0 s。Phase 4 测量层 + Doppler + FramePlane 缓存的实测开销让 P95 涨到 45.47 s（+9.5 %）。

**修订**：`test_baseline_sweep_200.m` C9 assert 从 45.0 → **47.0 s**（+13 %），`phase-4-measurement.md` §7 C9 + 顶层 audit §17.6 同步标注。47 s 包含：
- Phase 4 实测开销 +9.5 %（pwelch 多次调用 + `applyDopplerShift` complex exponential + FramePlane once-per-receiver 缓存）
- ~3 % 环境噪声头空间

**接手 AI 警告**：
- Phase 5 跑 1000 场景 MC 时若 P95 > 47 s 不要直接上修 budget，先 profile 找 hot spot。
- 测量层的 `pwelch` / `obw` 调用开销已被验证在主 hot path，进一步优化空间在 FramePlane 缓存复用 / pwelch window 复用。
- 跨机器变异 ~3 %，所以 Phase 5 必须在同一台机器或可控机器上跑 baseline。

### 4.3 annotation v2 schema 升顶（owner Q-A=A_full_replace）

## 5. 重要约定 / 反模式

### 5.1 强制约定（违反 = 回退）
1. **fail-fast > silent fallback**：错误 throw `CSRD:<Phase>:<Cause>` 异常码；进 `+csrd/+utils/+scenario/isScenarioSkipException.m` 白名单 → SkipScenario；不要 catch 后 silent log 继续。
2. **Truth.Design / Execution / Measured 三层铁律**：`Design` 来自蓝图，`Execution` 来自施工层，`Measured` 来自测量层；不要混。
3. **测量平面**：`SourcePlane`（信道后、合路前、receiver RF 链前 = `comp.Signal`，per-emitter）vs `FramePlane`（合路后、receiver RF 链前，per-receiver-frame）；不要从总接收信号反推 per-source GT。
4. **ReceiverView 投影**：emitter 对不同 receiver 的投影必须 receiver-view 化（5 字段），不要写成 emitter 全局字段。
5. **annotation 落盘前 hook**：`SimulationRunner.saveScenarioData` 内 `sanitizeForJson` → `validateMeasurementCompleteness` → `stampRuntimeHeader` 顺序不能改。
6. **MATLAB System object**：`setup → step → reset → release` 生命周期；channel block 在不同 frame 间不要共享状态除非物理意义如此。
7. **Random Seed**：deterministic replay；任何引入 stochastic branch 必须挂在已声明的 seed stream 上。
