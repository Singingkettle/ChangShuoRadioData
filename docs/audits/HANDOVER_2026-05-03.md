- 修复 `+csrd/+blocks/+physical/+txRadioFront/TRFSimulator.m`：

## 6. Phase 15：Building OSM RayTracing 与架构重整

核心问题：

- building OSM case 之前被 skip，不是因为 MATLAB 不能跑 OSM RayTracing，而是能力检测错了。
- `siteviewer` 在当前 runtime 可能是 `.p` 文件，`exist('siteviewer','file')` 返回 `6`，旧代码只接受 `2`，因此误判缺失。
- `raytrace` 是 `txsite` 方法路径，也不应按普通顶层函数检测。

修复：

- 新增 `+csrd/+runtime/+capabilities/rfPropagationCapabilities.m`
- 统一用 `which`、`exist(...,'class')` 和轻量 smoke probe 判断：
  - `siteviewer`
  - `txsite`
  - `rxsite`
  - `propagationModel`
  - `raytrace`
  - `comm.RayTracingChannel`
- building OSM 能力存在时不再错误 skip。
- empty/no-building OSM 保留显式 `FlatTerrain` fallback，且必须写 metadata/annotation。

参考资料：

- MathWorks `siteviewer` OSM buildings: https://www.mathworks.com/help/comm/ref/siteviewer.html
- MathWorks `raytrace`: https://www.mathworks.com/help/comm/ref/txsite.raytrace.html
- MathWorks `propagationModel("raytracing")`: https://www.mathworks.com/help/antenna/ref/propagationmodel.html
- MathWorks `comm.RayTracingChannel`: https://www.mathworks.com/help/comm/ref/comm.raytracingchannel-system-object.html

组织架构调整：

- `+csrd/+runtime`：toolbox/capability/sysinfo/logger/map runtime 支撑
- `+csrd/+catalog`：频谱 catalog、profile 数据
- `+csrd/+pipeline`：annotation、blueprint、contract、measurement、scenario helpers
- `+csrd/+support`：docs audit、validation、hash、random/optimization、path/introspection

接手注意：

## 10. 目前最可能的潜在 bug

1. **GT 时间框仍可能不准**
   - 原 annotation schema 主要描述频率 view，没有明确每个 source 的 frame-relative time bounds。
   - 已开始加 `StartTimeSec/EndTimeSec`，但需要验证 downstream converter、schema 文档、测量完整性 gate 是否接受。

2. **active interval 与 segment index 映射要确认**
   - `findOverlappingTransmissionIntervals` 现在裁剪 interval。
   - `buildSegmentConfigFromTxScenario` 用 `ActiveIntervalIndices` 找对应 `ActiveIntervals` 的位置。
   - 必须用多 interval 单帧 case 验证每个 segment 的 `segIdx`、时间、annotation 一一对应。

3. **接收端合成长度变了**
   - 修复 `comp.Signal(:)` 后，多天线 source 不再被错误拉长。
   - 这会改变多天线场景输出长度/能量，需要跑 `MergeChannelOutputContractTest`、`BaseChannelDistanceTest`、MIMO/RayTracing smoke。

4. **`TargetFrameSamples = 262144` 会显著放大运行时间**
   - 20 MHz 下单帧时长约 `0.0131072 s`。
   - 50 MHz 下单帧时长约 `0.00524288 s`。
   - 全量 69 case 高分辨率可能很慢。先单 case，再小矩阵，再全量。

5. **多 burst case 当前使用 Random pattern**
   - Random intervals 可能重叠或过宽，导致“多个 burst”视觉上像一大片。
   - 如果用户需要清楚的 on/off 多段 burst，建议新增 deterministic visual pattern 选项，直接写多个不重叠 interval，例如每个 Tx 三段短 burst，错开时间。

6. **OFDM 多天线模式需要继续压实**
   - `OSTBC` 和 `SpatialMultiplexing` 已显式化，但 pipeline annotation 只新增 `ModulationSpatialMode`，还没全面文档化。
   - 需要确认 `ModulationFactory` merge 后确实将 `ModulatorConfig.mimo.Mode` 送到 OFDM 对象。
